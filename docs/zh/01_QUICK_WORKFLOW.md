# 🔄 日常工作流：与 AI 智能体协作

> 在单个项目中协调多个 AI 工具的参考指南。

---

## 核心理念

每个智能体在开始工作前读取 `STATE.md`。每个智能体在停止前更新它。这就是整个系统。

---

## 日常工作流

### 开始一个会话
告诉你的 AI：
> *"读取 `.planning/STATE.md` 并从我们上次停下的地方继续。"*

### 工作过程中（使用 2 个智能体）

**[智能体 1] — 设计 / UI / 前端**
- 负责：UI 结构、CSS、布局、组件设计。
- 暂停前记录到 `STATE.md`：
  > *"已完成登录表单 UI。需要智能体 2 接入认证 API。"*

**[智能体 2] — 逻辑 / API / 后端**
- 负责：函数编写、数据处理、API 集成。
- 暂停前记录到 `STATE.md`：
  > *"认证 API 完成。智能体 1 现在可以连接表单了。"*

### 如果一个智能体额度耗尽
告诉剩余的智能体：
> *"你的搭档额度已耗尽。请阅读 `STATE.md` 并暂时接管他们的任务。"*

---

## 结束会话

关闭前，告诉 AI：
> *"请将今天完成的工作总结到 `$VIBE_HUB/SESSION_LOGS/latest.md`，以便下次继续。"*

---

## 提示

- 始终先让 AI 读取 `STATE.md`。这是最重要的习惯。
- `STATE.md` 中的任务描述要简短具体——模糊的交接说明会导致上下文混乱。
- 如果项目变得复杂，将 `STATE.md` 分阶段处理，并将旧阶段归档到 `STATE-archive.md`。
